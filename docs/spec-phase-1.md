# open-auggd — Project Specification

## Overview

`open-auggd` is a Python CLI utility that installs and manages an opinionated agentic
development workflow on top of [OpenCode](https://opencode.ai). It provides:

1. **Workflow enforcement** — a structured phase gate system (explore → plan → develop →
   review → finalize → document) with artifact prerequisites checked at each transition.
2. **Workspace management** — task-scoped artifact directories that carry context across
   agentic calls.
3. **OpenCode integration** — generates and manages agents, commands, skills, and custom
   tool wrappers inside `.opencode/` so the workflow is available inside an OpenCode session.

The CLI is installed as `auggd`. It is also exposed as a set of OpenCode custom tools
(`oag-<phase>_<action>`) so agents can call it programmatically during a session.

---

## Workflow Phases

```
explore → plan → develop → review → finalize → document
```

Each phase requires artifacts from the previous phase and produces its own artifacts. The
tool layer enforces these gates — an agent cannot call `develop start` if no approved plan
exists.

### Explore
- Ideation, research, external and internal fact-gathering.
- Can be re-entered at any point for consulting on ideas or reviewing outcomes.
- Produces: `explore/<topic>.md` (narrative), `explore/attachments.json` (registry).

### Plan
- Solution strategy based on exploration and prior iteration context.
- Each iteration must be a single TDD-sized task — no batching.
- Produces: `plan/spec.md` (workspace-wide spec), `plan/iter-N-plan.md` (narrative),
  `plan/iter-N-plan.json` (structured metadata), `plan/progress-log.json` (derived summary).

### Develop
- TDD implementation of exactly one planned iteration.
- Does not commit work.
- Produces: `develop/iter-N-devlog.md` (narrative), `develop/iter-N-devlog.json` (status).

### Review
- Internal gate: intent alignment + engineering smell detection.
- Produces: `review/iter-N-review.md` (narrative), `review/iter-N-review.json` (structured
  findings with severity labels).

### Finalize
- Used once a TDD cycle has an approved review.
- Marks iteration items, updates spec, commits changes.
- Replaces the "publish" concept — push/PR is a manual user decision after finalize.

### Document
- Project-wide scope (not workspace-scoped).
- Follows Diataxis categorization (tutorial / how-to / reference / explanation).
- Uses Mermaid syntax for diagrams.
- Tracks generation state in `.auggd/document-metadata.json` keyed to last git commit hash.
- Warns and exits if the project is not a git repository.

## Agents

### Primary

**Auggd** (`oag-auggd`)
- The main interfacing agent. Delegates to subagents based on user intent.
- Relays subagent outcomes back to the user without deep involvement in execution.
- Mode: `primary`.

### Subagents (all hidden from `@` autocomplete)

| Agent | Role | Adapted From |
|---|---|---|
| `oag-explorer` | Fact-gathering, research, ideation | researcher + parts of architect |
| `oag-planner` | Solution strategy, iteration shaping | architect (planning focus) |
| `oag-developer` | TDD implementation, devlog maintenance | developer |
| `oag-reviewer` | Intent + smell review, findings | reviewer |
| `oag-finalizer` | Iteration close-out, spec update, commit | manager (publish flow) |
| `oag-documenter` | Repo-level docs under `docs/` | documenter |

All subagents are `hidden: true`. All reference the `auggd tools *` commands for workflow
enforcement and the updated workspace artifact paths (`plan/spec.md`, not workspace root).

Agent prompts are adapted from the existing `opencode-setting-reference/` agents but updated
to reflect:
- New workspace directory structure
- `auggd tools *` enforcement layer
- Removal of trunk-based-specific constraints (general git workflow instead)
- `oag-` prefixed skill names

## Configuration

### Hierarchy (later overrides earlier)

```
.auggd/auggd.toml  <  OAG_* env vars  <  CLI flags
```

### Config File (`.auggd/auggd.toml`)

Generated by `auggd install`. Gitignored.

```toml
[workspace]
dir = ".auggd/workspace"   # supports ~ and $HOME expansion

[docs]
dir = "docs"

[models]
default = "opencode/gpt-5-nano"

[models.agents]
# Per-agent overrides. Key is agent name without "oag-" prefix.
# explorer = "anthropic/claude-haiku-4-5"
# developer = "anthropic/claude-haiku-4-5"

[models.commands]
# Per-command overrides. Key is command name without "oag-" prefix.
# explore = "anthropic/claude-haiku-4-5"
```

### Environment Variables

All prefixed with `OAG_`.

| Variable | Description | Default |
|---|---|---|
| `OAG_WORKSPACE_DIR` | Path to workspace root | `<project>/.auggd/workspace` |
| `OAG_DOCS_DIR` | Path to project docs | `<project>/docs` |
| `OAG_DEFAULT_MODEL` | Default model for all agents/commands | `opencode/gpt-5-nano` |
| `OAG_<AGENT>_MODEL` | Model override for a specific agent | — |
| `OAG_<COMMAND>_MODEL` | Model override for a specific command | — |

Path values support `~` and `$VARNAME` expansion.

Agent/command name in env var: uppercase, e.g. `OAG_EXPLORER_MODEL`, `OAG_DEVELOP_MODEL`.

### When Models Change

Run `auggd update` to regenerate the `model:` frontmatter field in all managed
`.opencode/agents/oag-*.md` and `.opencode/commands/oag-*.md` files. Content is not
modified — only the `model:` frontmatter line.

## Install / Uninstall Lifecycle

### `auggd install`

Run once per project from the project root.

```bash
cd path/to/project
pip install open-auggd
auggd install
```

Creates:
```

  auggd.toml                     # generated config
  install-manifest.json          # tracks all files written by auggd
  workspace/                     # workspace root (created on first ws create)
  document-metadata.json         # document phase metadata (created on first document init)

.opencode/                       # committed to git
  agents/
    oag-auggd.md
    oag-explorer.md
    oag-planner.md
    oag-developer.md
    oag-reviewer.md
    oag-finalizer.md
    oag-documenter.md
  commands/
    oag-explore.md
    oag-plan.md
    oag-develop.md
    oag-review.md
    oag-finalize.md
    oag-document.md
    oag-status.md
    oag-resume.md
  skills/
    oag-exploration-workflow/SKILL.md
    oag-planning-workflow/SKILL.md
    oag-development-workflow/SKILL.md
    oag-review-workflow/SKILL.md
    oag-finalize-workflow/SKILL.md
    oag-documentation-workflow/SKILL.md
    oag-tdd-standards/SKILL.md
    oag-documentation-standards/SKILL.md
    oag-devlog-template/SKILL.md
    oag-spec-template/SKILL.md
    oag-iteration-template/SKILL.md
    oag-review-template/SKILL.md
  tools/
    oag-explore.ts
    oag-plan.ts
    oag-develop.ts
    oag-review.ts
    oag-finalize.ts
    oag-document.ts
```

The `install-manifest.json` records every file path written, enabling clean uninstall.

### `auggd uninstall`

```bash
auggd uninstall
# Removes .auggd/ entirely
# Removes all files listed in install-manifest.json from .opencode/
```

Only files created by `auggd install` are removed. Pre-existing `.opencode/` content is
untouched.

### `auggd update`

Updates the `model:` frontmatter in all managed agent and command files to reflect current
config/env settings. Does not touch content body or other frontmatter fields.

### `auggd reset`

Restores all `.opencode/` files managed by auggd to their bundled defaults.
`.auggd/auggd.toml` and workspace data are not affected.

Requires exact confirmation: `Type 'yes' to confirm:`.

## Workspace Management

### ID Format

Each workspace has a unique ID:

```
<uuid7-base64url-22chars>-<normalized-slug>
```

Example: `AZXyo7HEfomfPS0bTF1uf-add-user-authentication`

UUID7 is time-ordered, so workspaces sort naturally by creation time without an index file.

### Slug Normalization

Input slug is normalized to:
1. Unicode → ASCII (NFKD decomposition, non-ASCII stripped)
2. Lowercase
3. Non-alphanumeric → `-`
4. Consecutive `-` collapsed to single `-`
5. Leading/trailing `-` stripped
6. Truncated to 100 characters at a word boundary
7. Empty result raises `ValueError`

### Workspace Directory Structure

```
<workspace-id>/
  explore/
    <topic>.md               # exploration narrative (agent-written)
    attachments.json         # phase registry: status, topics, sources, attachments
  plan/
    spec.md                  # workspace-wide spec (lives here, not workspace root)
    iter-N-plan.md           # iteration narrative (agent-written)
    iter-N-plan.json         # structured metadata: status, acceptance_criteria, scope
    progress-log.json        # derived summary of all iteration statuses
  develop/
    iter-N-devlog.md         # devlog narrative (agent-written)
    iter-N-devlog.json       # status, files_touched, tests_run, next_red_step, blockers
  review/
    iter-N-review.md         # review narrative (agent-written)
    iter-N-review.json       # status, findings [{severity, file, description, suggestion}]
  tmp/                       # scratch space
```

Iteration numbers are globally monotonic per workspace (`iter-1`, `iter-2`, ...). A fix
cycle produces `iter-4`, not a new `iter-1`.

### Commands

```bash
auggd ws create <task-slug>   # create workspace
auggd ws list                 # list all workspaces with numbers
auggd ws delete <N>           # hard delete with y/N confirmation
auggd ws info <N>             # show spec description, topics, iteration summary
```

#### `auggd ws list` output format

```
 #  Name                             Phase      Iters
 1  add-user-authentication          develop    3 planned, 2 dev, 1 reviewed
 2  fix-auth-middleware              plan       1 planned
```

Numbers are positional (sorted by UUID7 = creation time). After a delete, numbers
re-index on the next `list` call.

#### Workspace reference in tool commands

`--ws` accepts either the list number or a slug substring (unambiguous match):

```bash
auggd tools develop --ws=1 start 2
auggd tools develop --ws=fix-auth start 2
```

## Artifact Schemas

All JSON artifacts use ISO 8601 timestamps. All Markdown narratives are agent-written and
not validated for content, only for existence in Phase 1. There will be a planned Phase 2
where narrative content is more strictly templated and validated.


### `explore/attachments.json`

```json
{
  "explore_status": "pending | in_progress | done",
  "topics": ["<filename without .md>"],
  "sources": [{"url": "", "title": "", "relevance": ""}],
  "attachments": [{"filename": "", "description": ""}],
  "updated_at": "2026-03-10T00:00:00Z"
}
```

### `plan/iter-N-plan.json`

```json
{
  "n": 1,
  "status": "pending | in_progress | dev_complete | review_complete | finalized",
  "title": "",
  "acceptance_criteria": [],
  "scope": [],
  "non_goals": [],
  "created_at": "2026-03-10T00:00:00Z",
  "updated_at": "2026-03-10T00:00:00Z"
}
```

### `plan/progress-log.json`

Derived. Written by `finalize`. When stale or absent, tools derive status from individual
`iter-N-plan.json` files directly and warn.

```json
{
  "workspace_id": "<full-workspace-dir-name>",
  "iterations": [
    {
      "n": 1,
      "plan_status": "pending",
      "dev_status": null,
      "review_status": null,
      "finalized": false,
      "finalized_at": null
    }
  ],
  "updated_at": "2026-03-10T00:00:00Z"
}
```

### `develop/iter-N-devlog.json`

```json
{
  "n": 1,
  "status": "in_progress | dev_complete",
  "files_touched": [],
  "tests_run": [{"command": "", "result": ""}],
  "blockers": [],
  "next_red_step": "",
  "updated_at": "2026-03-10T00:00:00Z"
}
```

### `review/iter-N-review.json`

```json
{
  "n": 1,
  "status": "blocked | changes_requested | approved",
  "findings": [
    {
      "severity": "MUST-FIX | SHOULD-FIX | NICE-TO-HAVE",
      "file": "",
      "description": "",
      "suggestion": ""
    }
  ],
  "updated_at": "2026-03-10T00:00:00Z"
}
```

### `.auggd/document-metadata.json`

```json
{
  "last_commit_hash": "abc1234",
  "last_commit_summary": "",
  "generated_at": "2026-03-10T00:00:00Z",
  "documents_updated": [],
  "known_gaps": []
}
```

## Tools Layer

### Registration in OpenCode

Each phase has a `.opencode/tools/oag-<phase>.ts` file with named exports. OpenCode
registers them as `oag-<phase>_<action>` tools.

Example (`oag-explore.ts` exposes `oag-explore_start`, `oag-explore_done`,
`oag-explore_status`).

Each TypeScript wrapper calls the Python CLI via `Bun.$`:

```typescript
export const start = tool({
  description: "Start explore phase. Creates explore/ structure and attachments registry.",
  args: { ws: tool.schema.string().describe("Workspace number or slug") },
  async execute(args) {
    return await Bun.$`auggd tools explore --ws=${args.ws} start`.text()
  },
})
```

Tool wrappers are committed to git. `.auggd/` is gitignored.

### Tool Response Format

All tool actions return JSON to stdout, exit 0. Errors are in the payload, not exit codes,
so OpenCode captures the full message.

```json
// Success
{ "ok": true, "data": { ... } }

// Failure
{
  "ok": false,
  "error": "MISSING_PLAN",
  "message": "No iter-1-plan.json found. Run plan first.",
  "missing": ["plan/iter-1-plan.json"]
}
```

### Action Matrix

| Phase | Action | Prerequisite | Effect |
|---|---|---|---|
| `explore` | `start` | workspace exists | creates `attachments.json` if absent |
| `explore` | `status` | — | reads `attachments.json` |
| `explore` | `done` | `attachments.json` exists | sets `explore_status = done` |
| `plan` | `start` | explore done | creates `plan/` subdirs, `progress-log.json` |
| `plan` | `iter create` | plan started | creates `iter-N-plan.{md,json}`, updates progress-log |
| `plan` | `iter status <N>` | iter exists | reads `iter-N-plan.json` |
| `plan` | `iter done <N>` | acceptance_criteria non-empty | sets status to `pending` (ready for dev) |
| `develop` | `start <N>` | `iter-N-plan.json` exists and is ready | creates devlog files, sets `in_progress` |
| `develop` | `update <N>` | devlog exists | merges provided JSON fields into devlog |
| `develop` | `status <N>` | — | reads `iter-N-devlog.json` |
| `develop` | `done <N>` | devlog exists | sets `dev_complete` |
| `review` | `start <N>` | `dev_complete` | creates `iter-N-review.{md,json}` |
| `review` | `update <N>` | review started | merges findings into review JSON |
| `review` | `done <N>` | findings written | sets review status |
| `finalize` | `iter <N>` | review `approved` | sets finalized, updates progress-log, optionally commits |
| `document` | `check` | — | verifies git repo, returns current hash and metadata status |
| `document` | `init` | git repo | creates `.auggd/document-metadata.json` skeleton |
| `document` | `update-metadata` | metadata exists | writes new commit hash + summary |

### CLI Interface for Tools

```bash
# Phase + workspace + action
auggd tools explore --ws=<N|slug> start
auggd tools explore --ws=<N|slug> done
auggd tools explore --ws=<N|slug> status

auggd tools plan --ws=<N|slug> start
auggd tools plan --ws=<N|slug> iter create
auggd tools plan --ws=<N|slug> iter status <N>
auggd tools plan --ws=<N|slug> iter done <N>

auggd tools develop --ws=<N|slug> start <N>
auggd tools develop --ws=<N|slug> update <N> --data '<json>'
auggd tools develop --ws=<N|slug> status <N>
auggd tools develop --ws=<N|slug> done <N>

auggd tools review --ws=<N|slug> start <N>
auggd tools review --ws=<N|slug> update <N> --data '<json>'
auggd tools review --ws=<N|slug> done <N>

auggd tools finalize --ws=<N|slug> iter <N>

auggd tools document check
auggd tools document init
auggd tools document update-metadata --data '<json>'
```

10. `templates/` — placeholder files to be fleshed out in Phase 2

## Out of Scope for Phase 1

- Actual agent/command/skill content (placeholder files only in `templates/`)
- `auggd tools document` full git integration (basic git presence check only)
- `opencode.json` modifications (agents installed as `.md` files only)
- Multi-user or team workspace sharing
